# UI GUIDELINES

## Design Intent

- Industrial SCADA-like visual language, not SaaS-template aesthetics
- Dark-first interface with restrained electric blue and cyan accents
- Glassmorphism with readability-first contrast and thin borders

## Rules

- No fake KPIs or placeholder analytics in production views
- Every dashboard panel must map to a real query or computed metric
- Keep transitions subtle; avoid distracting or high-amplitude animations
- Always include loading/error empty-state messaging

## Reuse Pattern

- Build screens from reusable cards, badges, inputs, tables, dialogs, and charts
- Keep forms machine-centric and parameter-driven
- Maintain compact data density for operator workflows
